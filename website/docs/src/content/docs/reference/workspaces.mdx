`agentsync` is a multi-language project that combines a high-performance Rust core with a TypeScript distribution layer. This polyglot architecture ensures maximum performance where it matters, while providing a seamless installation experience for the JavaScript ecosystem.

## Repository Structure

- `src/`: Core logic and CLI implementation in **Rust**. This is the heart of AgentSync.
- `npm/agentsync/`: **TypeScript** wrapper used for NPM distribution. It handles binary resolution for different platforms.
- `website/docs/`: Documentation site built with **Starlight**.
- `tests/`: Integration tests for the CLI, including E2E tests.

---

## Rust Workspace (The Core)

The core logic is written in Rust for speed, safety, and easy distribution as a single binary.

- **Tools**: Cargo, Make
- **Version**: Rust 1.89+ (Edition 2024)
- **Key Commands**:

  ```bash
  cargo build    # Build debug binary
  cargo test     # Run unit and integration tests
  cargo run      # Run the CLI locally
  ```

- **Error Handling**: Uses `thiserror` for custom errors and `anyhow` for application-level error handling.

---

## JavaScript Workspace (Distribution)

The JS workspace acts as a wrapper to provide an easy installation path via `pnpm/npm`. It manages platform-specific binary downloads.

- **Tools**: pnpm, TypeScript, Make
- **Location**: `npm/agentsync/`
- **Key Commands**:

  ```bash
  pnpm run typecheck  # Verify types
  pnpm run build      # Compile TypeScript to JS
  ```

- **Strictness**: We enforce strict TypeScript configurations. Avoid `any` at all costs.

---

## Cross-Stack Workflow

The mono-repo is designed for tight integration between Rust and TypeScript:

1.  **Development**: Core features are implemented in Rust (`src/`).
2.  **Wrapper Sync**: The TypeScript wrapper (`npm/agentsync/`) is kept in sync with the core's version and CLI API.
3.  **Unified Build**: We use a central `Makefile` to ensure consistent builds across environments.

## Modern Tooling

We use a **Makefile** to orchestrate tasks across both Rust and JavaScript stacks.

- **`make install`**: Bootstraps the development environment. Runs `pnpm install` and `cargo build`.
- **`make verify-all`**: Our CI-in-a-box command. Runs all tests (JS + Rust), linters (Biome + Clippy), and builds the documentation.
- **`make rust-test`**: Runs all Rust unit and integration tests.
- **`make js-test`**: Runs type checks for the TypeScript wrapper.
- **`make js-build`**: Compiles TypeScript and prepares the NPM package.
- **`make docs-build`**: Verifies that the documentation site builds correctly.
- **`make fmt`**: Auto-formats the entire repository using `rustfmt` and `biome`.
- **`make clean`**: Removes build artifacts (`target/`, `dist/`, etc.).

We also recommend these modern CLI tools:
- `bat`: Syntax-highlighted file viewing (replaces `cat`).
- `rg` (ripgrep): Fast recursive search (replaces `grep`).
- `fd`: Simple and fast alternative to `find`.
- `sd`: Intuitive find and replace (replaces `sed`).
- `eza`: Feature-rich directory listing (replaces `ls`).
